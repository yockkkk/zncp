# 智能餐饮推荐系统开发文档（本地开发版）

## 版本说明

本文档用于指导开发一套基于“图片上传 + 多模态大模型分析 + 向量数据库语义召回 + MySQL 数据查询 + 大模型重排序”的智能餐饮推荐系统。

当前版本**不考虑部署**，只关注本地开发和本地联调。

建议先完成最小可行版本，也就是：

```text
图片上传 → OSS/本地Mock地址 → AI分析用户画像 → 生成推荐查询文本 → Embedding向量化 → Qdrant召回菜品 → MySQL查询菜品详情 → 大模型排序 → 前端展示推荐结果
```

视频推荐、用户登录、订单系统、历史记录、营养报表等功能可以作为后续扩展。

---

# 一、项目目标

## 1.1 项目名称

基于多模态大模型与向量数据库的智能餐饮推荐系统。

## 1.2 项目核心功能

本系统需要实现以下核心能力：

1. 用户可以在网页端上传顾客图片。
2. 后端将图片上传至 OSS，或者本地开发阶段先生成一个 mock 图片地址。
3. 后端调用多模态大模型分析图片内容。
4. 大模型返回顾客画像，例如用餐人数、用餐场景、消费层级、偏好标签等。
5. 系统将顾客画像转换成推荐查询文本。
6. 系统调用 Embedding 模型生成查询向量。
7. 系统使用 Qdrant 向量数据库召回相似菜品。
8. 系统根据召回结果到 MySQL 查询菜品详情。
9. 系统使用规则过滤和大模型重排序生成最终推荐结果。
10. 前端展示推荐菜品、推荐理由、营养信息、性价比说明等。

## 1.3 项目技术栈

| 模块 | 技术选型 |
|---|---|
| 前端 | Vue 3 + Vite + Element Plus |
| 后端 | Spring Boot |
| 业务数据库 | MySQL |
| 缓存 | Redis，可选 |
| 文件存储 | 阿里云 OSS，本地开发可先用 mock 地址 |
| 向量数据库 | Qdrant |
| AI 能力 | 多模态大模型 + Embedding 模型 |

# 五、数据库设计

## 5.1 创建数据库

```sql
CREATE DATABASE food_recommend DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE food_recommend;
```

## 5.2 菜品表 dish

```sql
CREATE TABLE dish (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '菜品ID',
    name VARCHAR(100) NOT NULL COMMENT '菜品名称',
    category VARCHAR(50) COMMENT '菜品分类',
    price DECIMAL(10,2) COMMENT '价格',
    calories INT COMMENT '热量，单位kcal',
    protein DECIMAL(10,2) COMMENT '蛋白质，单位g',
    fat DECIMAL(10,2) COMMENT '脂肪，单位g',
    carbohydrate DECIMAL(10,2) COMMENT '碳水，单位g',
    taste VARCHAR(100) COMMENT '口味，例如清淡、麻辣、酸甜',
    suitable_people VARCHAR(255) COMMENT '适合人群',
    scene VARCHAR(255) COMMENT '推荐场景',
    tags VARCHAR(255) COMMENT '菜品标签',
    image_url VARCHAR(500) COMMENT '菜品图片',
    description TEXT COMMENT '菜品描述',
    sales INT DEFAULT 0 COMMENT '销量',
    stock INT DEFAULT 999 COMMENT '库存',
    status TINYINT DEFAULT 1 COMMENT '状态：1上架，0下架',
    vector_status TINYINT DEFAULT 0 COMMENT '向量状态：0未生成，1已生成',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT='菜品表';
```

## 5.3 推荐记录表 recommendation_record


# 七、Qdrant 向量数据库开发

## 7.1 Qdrant 的作用

Qdrant 用来存储菜品的语义向量，并根据用户画像向量召回最相似的菜品。

MySQL 存真实业务数据：

```text
菜品名称
价格
热量
蛋白质
库存
销量
上下架状态
```

Qdrant 存语义检索数据：

```text
菜品向量
菜品标签
菜品分类
适合人群
推荐场景
```

## 7.2 本地启动 Qdrant

如果你本地安装了 Docker，可以执行：

```bash
docker run -d \
  --name food-qdrant \
  -p 6333:6333 \
  -v ./qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

启动后访问：

```text
http://localhost:6333
```

如果你暂时不想用 Docker，也可以先把 VectorSearchService 写成 mock，等主流程跑通后再接入 Qdrant。

## 7.3 Collection 设计

Collection 名称：

```text
dish_vector_collection
```

向量维度要和你使用的 Embedding 模型一致。

例如：

```text
如果 Embedding 模型输出 1536 维，则 Qdrant collection vector size = 1536。
如果 Embedding 模型输出 1024 维，则 Qdrant collection vector size = 1024。
```

## 7.4 Qdrant 中的 payload 设计

```json
{
  "dishId": 1001,
  "name": "鸡胸肉沙拉",
  "category": "减脂餐",
  "price": 28,
  "calories": 320,
  "protein": 35,
  "taste": "清淡",
  "tags": ["低脂", "高蛋白", "减脂", "健身"],
  "suitablePeople": ["健身人群", "减脂人群", "年轻用户"],
  "scene": ["午餐", "晚餐", "训练后"]
}
```

# 八、Embedding 文本设计

## 8.1 菜品向量文本

不要只使用菜品名称生成向量。

错误示例：

```text
鸡胸肉沙拉
```

推荐示例：

```text
菜品名称：鸡胸肉沙拉。
分类：减脂餐。
价格：28元。
营养信息：热量320千卡，蛋白质35克，脂肪8克，碳水20克。
口味特点：清淡、爽口。
适合人群：健身人群、减脂人群、年轻用户。
推荐场景：午餐、晚餐、训练后补充蛋白质。
推荐理由：低脂高蛋白，适合控制体重和补充蛋白质。
```

## 8.2 后端拼接方法示例

```java
public String buildDishEmbeddingText(Dish dish) {
    return "菜品名称：" + dish.getName() + "。" +
            "分类：" + dish.getCategory() + "。" +
            "价格：" + dish.getPrice() + "元。" +
            "营养信息：热量" + dish.getCalories() + "千卡，蛋白质" + dish.getProtein() + "克，脂肪" + dish.getFat() + "克，碳水" + dish.getCarbohydrate() + "克。" +
            "口味特点：" + dish.getTaste() + "。" +
            "适合人群：" + dish.getSuitablePeople() + "。" +
            "推荐场景：" + dish.getScene() + "。" +
            "菜品标签：" + dish.getTags() + "。" +
            "菜品描述：" + dish.getDescription();
}
```

## 8.3 用户画像查询文本

AI 分析图片后，会返回用户画像 JSON。

后端要把 JSON 转成一段推荐查询文本。

示例：

```text
当前有2人用餐，用户年龄段约20-30岁，场景为朋友聚餐，消费能力中等，偏好性价比、营养均衡和清淡口味，适合推荐价格适中、分量合适、营养搭配均衡的菜品组合。
```

Java 示例：

```java
public String buildRecommendQueryText(UserProfileDTO profile) {
    return "当前有" + profile.getPeopleCount() + "人用餐，" +
            "用户年龄段约" + profile.getAgeRange() + "，" +
            "场景为" + profile.getDiningScene() + "，" +
            "消费能力" + profile.getEstimatedConsumptionLevel() + "，" +
            "偏好" + String.join("、", profile.getPossiblePreferences()) + "，" +
            "健康目标为" + profile.getHealthGoal() + "，" +
            "适合推荐价格合理、营养搭配合适、符合当前场景的菜品。";
}
```

# 九、AI 提示词设计

## 9.1 图片分析提示词

保存到 prompt_template 表中，code 可以设置为：

```text
VISION_ANALYZE_CUSTOMER
```

内容：

```text
你是一个餐饮推荐系统中的顾客场景分析助手。

请根据用户上传的图片进行非身份识别式场景分析，不要识别具体身份，不要输出真实姓名，不要做敏感身份判断。

请分析以下内容：
1. 图片中大概有几人用餐
2. 当前可能的用餐场景
3. 用户可能偏好的餐饮类型
4. 可能的消费能力等级：低 / 中 / 高
5. 是否适合推荐减脂餐、营养餐、套餐、多人餐
6. 推荐关键词

请严格按照 JSON 格式输出：

{
  "peopleCount": 2,
  "ageRange": "20-30",
  "diningScene": "朋友聚餐",
  "clothingStyle": "休闲",
  "estimatedConsumptionLevel": "中等",
  "possiblePreferences": ["性价比", "营养均衡", "清淡"],
  "healthGoal": "日常均衡饮食",
  "recommendationKeywords": ["两人餐", "价格适中", "营养均衡"]
}
```

## 9.2 菜品重排序提示词

code 可以设置为：

```text
RERANK_DISH_RECOMMEND
```

内容：

```text
你是一个智能餐饮推荐专家。
```

# 十二、核心业务流程伪代码

## 12.1 图片推荐主流程

```java
public RecommendResultDTO recommendByImage(MultipartFile file, Long userId) {

    // 1. 上传图片，得到图片 URL。本地开发可先返回 mock URL
    String imageUrl = ossService.uploadFile(file);

    // 2. 调用多模态大模型分析图片
    UserProfileDTO profile = visionModelService.analyzeImage(imageUrl);

    // 3. 根据用户画像生成推荐查询文本
    String queryText = buildRecommendQueryText(profile);

    // 4. Qdrant 召回相似菜品 ID
    List<Long> dishIds = vectorSearchService.searchSimilarDishes(queryText, 20);

    // 5. 从 MySQL 查询菜品详情
    List<Dish> candidateDishes = dishMapper.selectByIds(dishIds);

    // 6. 规则过滤
    List<Dish> filteredDishes = ruleFilter(candidateDishes, profile);

    // 7. 调用大模型重排序并生成推荐理由
    List<RecommendDishDTO> finalResults = aiRerankService.rerank(profile, filteredDishes);

    // 8. 保存推荐记录
    Long recordId = saveRecommendationRecord(userId, imageUrl, profile, queryText, finalResults);

    // 9. 返回结果
    RecommendResultDTO result = new RecommendResultDTO();
    result.setRecordId(recordId);
    result.setImageUrl(imageUrl);
    result.setUserProfile(profile);
    result.setRecommendations(finalResults);
    return result;
}
```

## 12.2 规则过滤逻辑

```java
public List<Dish> ruleFilter(List<Dish> dishes, UserProfileDTO profile) {
    return dishes.stream()
            .filter(dish -> dish.getStatus() == 1)
            .filter(dish -> dish.getStock() != null && dish.getStock() > 0)
            .filter(dish -> priceMatch(dish, profile))
            .sorted((a, b) -> b.getSales().compareTo(a.getSales()))
            .limit(10)
            .collect(Collectors.toList());
}
```

## 12.3 价格匹配逻辑示例

```java
private boolean priceMatch(Dish dish, UserProfileDTO profile) {
    String level = profile.getEstimatedConsumptionLevel();
    BigDecimal price = dish.getPrice();

    if ("低".equals(level)) {
        return price.compareTo(new BigDecimal("30")) <= 0;
    }
    if ("中等".equals(level)) {
        return price.compareTo(new BigDecimal("80")) <= 0;
    }
    return true;
}
```

# 十三、推荐评分设计

系统可以使用如下评分公式：

```text
最终得分 = 语义相似度 × 0.35
        + 营养匹配度 × 0.20
        + 价格匹配度 × 0.15
        + 场景匹配度 × 0.15
        + 销量评分 × 0.10
        + 库存状态 × 0.05
```

## 13.1 指标说明

| 指标 | 说明 |
|---|---|
| 语义相似度 | Qdrant 返回的相似度分数 |
| 营养匹配度 | 是否符合减脂、高蛋白、均衡等目标 |
| 价格匹配度 | 是否符合消费能力 |
| 场景匹配度 | 是否适合单人餐、双人餐、聚餐 |
| 销量评分 | 热门菜品适当加权 |
| 库存状态 | 有库存才推荐 |

MVP 阶段可以先不手写复杂评分，先用：

```text
Qdrant 召回 + 简单规则过滤 + 大模型重排序
```

后期再补充评分公式。

# 十九、答辩介绍词

可以这样介绍系统：

```text
本系统是一套基于多模态大模型与向量数据库的智能餐饮推荐系统。用户上传图片后，系统首先获取图片地址，然后调用多模态大模型进行非身份识别式场景分析，生成用餐人数、用餐场景、消费层级和偏好标签等用户画像。

系统会将用户画像转换成推荐查询文本，并通过 Embedding 模型转成语义向量，再到 Qdrant 向量数据库中召回最相似的真实菜品。之后系统根据 MySQL 中的价格、热量、蛋白质、库存、销量等结构化数据进行过滤，最后由大模型完成重排序并生成推荐理由。

该系统相比普通关键词推荐，能够更好理解用户场景和语义需求；相比直接让大模型生成菜品，又能保证推荐结果来自真实菜品库，降低大模型幻觉问题。
```
