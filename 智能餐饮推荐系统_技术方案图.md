## 图三：菜品向量预生成流程（提前执行一次）

```
MySQL dish 表（50+ 条菜品）
          │
          ▼
  遍历每道菜品
          │
          ▼
┌─────────────────────────────────┐
│  拼接菜品语义描述文本              │
│  "菜品名称:鸡胸肉沙拉。分类:减脂餐。│
│   价格:28元。热量:320千卡。        │
│   蛋白质:35克。适合人群:健身人群…" │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  text-embedding-v3              │
│  文本 → 1024维浮点数向量           │
│  [0.023, -0.014, 0.057, ...]    │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  写入 Qdrant                     │
│  id = 菜品ID                     │
│  vector = 1024维向量              │
│  payload = 菜品标签/分类/场景等    │
└──────────────┬──────────────────┘
               │
               ▼
  MySQL dish.vector_status = 1
  （标记该菜品向量已生成）
```

**调用方式：**
```
POST http://localhost:8080/api/admin/dish/vector/batch-rebuild
```
